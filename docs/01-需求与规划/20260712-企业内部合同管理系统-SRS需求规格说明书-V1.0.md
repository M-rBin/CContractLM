## 二、项目概述

### 2.1 项目背景

随着企业经营规模扩大，对外签订的合同数量持续增长，合同管理长期依赖分散文件、线下表格和人工提醒，暴露出以下问题：

1. **合同信息散落难以追溯**：合同正文、补充协议、验收材料分别保存在个人电脑、聊天记录和纸质档案中，缺少统一台账，查找历史合同和相对方信息耗时长，容易遗漏。
2. **履约节点跟踪缺失**：交付、验收、续签、终止等关键事项依赖负责人个人记忆和手工记录，节点临近或超期时缺乏系统提醒，履约遗漏和违约风险高。
3. **收付款进度不透明**：合同的收款和付款计划分散在多份表格中，待收、待付、逾期金额无法实时汇总，财务和业务之间信息不同步，容易出现漏收漏付。
4. **到期与逾期提醒依赖人工**：合同到期、节点到期、收付款到期均靠人工翻查台账提醒，一旦经办人变动或遗忘，事项无人跟进。
5. **经营情况缺乏整体视图**：管理者难以快速掌握合同规模、金额分布、收付款进度和逾期风险，缺少辅助决策的数据支撑。

为解决上述问题，建设企业内部合同管理系统，将合同从签署后的登记、归档、履约跟踪、收付款跟进到到期提醒和经营分析集中到统一系统中管理，提升合同管理的规范性、及时性和可追溯性。

### 2.2 项目建设目标

本项目旨在建设一套面向企业内部的合同集中管理系统，实现以下目标：

1. **建立统一合同台账**：集中登记和维护合同基础信息，支持多维度查询、筛选和导出，减少合同信息散落在个人电脑、聊天记录和表格中的情况，形成可追溯的合同主数据。
2. **实现多类型附件归档**：支持合同正文、补充协议、验收材料、发票、付款凭证等多类型附件按业务分类上传、查看、下载和删除，为合同全生命周期材料提供集中存档。
3. **强化履约节点跟踪**：为每份合同维护多个履约节点，记录交付、验收、续签、到期、终止等关键事项，根据计划日期判断临近和逾期状态，减少履约事项遗漏。
4. **跟踪收付款计划**：为每份合同维护多期收款或付款计划，记录计划金额、计划日期、实际金额和实际日期，形成待收、待付、已收、已付、逾期等状态管理，提升收付款跟进效率。
5. **提供站内提醒与工作台**：根据合同到期日、履约节点计划日期和收付款计划日期生成站内提醒，并在登录后默认工作台集中展示关键指标、待办提醒和看板图表，帮助用户快速总览和处理待办。
6. **辅助经营分析决策**：汇总合同规模、金额分布、收付款进度和逾期风险，为管理者了解合同整体情况和履约收付款风险提供数据支撑。

### 2.3 项目建设范围

本项目建设范围包括：

**系统端**：

- PC 端管理后台（Web 应用），基于企业已有的管理端和服务端能力扩展，复用用户、角色、部门、菜单、权限、日志等基础模块。

**功能模块**：

- 合同工作台：登录后默认首页，总览关键指标、待办提醒、看板图表、快捷入口和最近动态。
- 合同台账：合同的登记、查询、筛选、编辑、查看、删除和导出。
- 合同附件：合同附件按业务材料类型分类上传、查看、下载和删除。
- 履约节点：为合同维护多个履约节点，跟踪交付、验收、续签、终止等关键事项。
- 收付款计划：为合同维护多期收款或付款计划，跟踪计划与实际收付款。
- 站内提醒：根据合同到期、履约节点和收付款计划生成站内提醒，集中展示并支持跳转来源。
- 经营分析：展示合同经营指标、金额分布、收付款进度和逾期风险。

**数据范围**：

- 合同基础信息、合同附件、履约节点、收付款计划、站内提醒和合同操作记录。
- 合同数据第一版对所有登录用户可见，不按部门、负责人或业务线隔离。

**不包含的功能**：

- 线上审批流：合同内部审批在线下完成后在系统中登记，本期不建设在线审批路径（后续迭代实现）。
- 合同在线编辑与电子签章：本期不提供合同正文在线起草和电子签章能力。
- 附件版本管理与在线预览强依赖：附件本期只做分类归档，不做历史版本追踪和在线预览。
- 外部通知渠道：本期只做站内提醒，不接入邮件、短信、企业微信、钉钉等外部渠道。
- 财务系统对接：合同金额只做记录和统计，不做发票核销、银行流水匹配和复杂汇率换算。
- 复杂权限隔离：本期合同数据对所有登录用户可见，不按数据范围细分可见性。

## 三、项目建设内容

### 3.1 总体功能架构

![](images/system-architecture.png)

系统在企业已有管理端的基础上新增“合同管理”能力，并复用基础的系统配置模块。菜单结构如下：

- 工作台（一级菜单）
- 合同管理（一级菜单）
  - 合同台账（二级菜单）
  - 履约提醒（二级菜单）
  - 收付款管理（二级菜单）
  - 经营分析（二级菜单）
- 系统配置（一级菜单）
  - 用户管理（二级菜单）
  - 角色管理（二级菜单）
  - 部门管理（二级菜单）
  - 岗位管理（二级菜单）
  - 字典管理（二级菜单）
  - 操作日志（二级菜单）

用户登录后默认进入工作台。合同管理是本系统的核心，工作台汇总合同关键指标与待办；合同台账是合同登记与查询的主入口，合同详情从台账进入，用于集中维护基础信息、附件、履约节点、收付款计划和操作记录；履约提醒和收付款管理分别从履约与收付款维度集中跟进；经营分析提供整体经营视图。系统配置复用已有的用户、角色、部门、岗位、字典和日志能力，为合同管理提供权限与基础数据支撑。

### 3.5 功能详细设计

### 3.5.1 合同工作台

**（一）功能需求概述**

合同工作台是登录后的默认首页，用于集中展示合同履行、收付款相关的关键数据和待办事项，帮助用户快速掌握整体业务状况并直达高频操作入口。工作台本身不维护业务数据，所有内容均实时汇总自合同台账、履约节点、收付款计划、站内提醒等模块。

**（二）功能设计**

**关键指标**

展示以下 7 项关键指标：

| 指标名称 | 说明 |
|---------|------|
| 合同总数 | 系统内全部合同的数量，来源于合同台账 |
| 履行中合同 | 合同状态为"履行中"的合同数量，来源于合同台账 |
| 即将到期合同 | 到期日期在临近阈值范围内且合同状态未终止/归档的合同数量，来源于合同台账 |
| 逾期合同 | 已过到期日期且合同状态未变更为已终止或已归档的合同数量，来源于合同台账 |
| 待收金额 | 收款方向的收付款计划中状态为"待处理"的计划金额合计，来源于收付款管理数据 |
| 待付金额 | 付款方向的收付款计划中状态为"待处理"的计划金额合计，来源于收付款管理数据 |
| 逾期金额 | 状态为"逾期"的收付款计划金额合计，来源于收付款管理数据 |

**待办提醒**

展示以下 4 类待办提醒，来源于站内提醒：

| 提醒类型 | 说明 |
|---------|------|
| 合同到期提醒 | 合同到期日期临近或已到期的提醒 |
| 履约节点提醒 | 履约节点计划日期临近或已逾期的提醒 |
| 收款计划提醒 | 收款计划日期临近或已逾期的提醒 |
| 付款计划提醒 | 付款计划日期临近或已逾期的提醒 |

点击某条待办提醒，跳转至对应处理位置：合同到期提醒跳转至合同详情，履约节点提醒跳转至履约提醒，收款计划提醒和付款计划提醒跳转至收付款管理。

**看板图表**

展示以下 4 项看板图表，来源于合同台账、收付款管理数据：

| 图表名称 | 说明 |
|---------|------|
| 合同状态分布 | 按合同状态（草稿/履行中/已到期/已终止/已归档）查看各状态合同数量占比 |
| 合同类型分布 | 按合同类型（采购合同/销售合同/服务合同/租赁合同/框架协议/其他合同）查看各类型合同数量占比 |
| 月度签约金额 | 按月查看签约金额变化趋势 |
| 收付款完成情况 | 查看收付款计划的完成进度 |

**快捷入口**

展示以下 4 个快捷入口：

| 入口名称 | 说明 |
|---------|------|
| 新增合同 | 跳转至新增合同页面 |
| 合同台账 | 跳转至合同台账列表 |
| 履约提醒 | 跳转至履约节点提醒列表 |
| 收付款管理 | 跳转至收付款管理列表 |

**最近动态**

展示以下 3 项最近动态，均按时间倒序取前若干条：

| 动态名称 | 说明 |
|---------|------|
| 最近新增合同 | 最近新增的合同记录，来源于合同台账 |
| 最近完成履约节点 | 最近标记完成的履约节点记录，来源于履约节点数据 |
| 最近完成收付款记录 | 最近登记完成的收付款记录，来源于收付款计划数据 |

点击某条最近动态，跳转至对应的合同详情。

**业务规则**

- 即将到期/临近的判断默认取到期日期（或计划日期）前 30 天，可由系统参数调整。
- 逾期判断基于计划日期和完成状态综合判定，已标记完成的履约节点或收付款计划不再判定为逾期。
- 合同到期不自动等同于合同终止，工作台仅提示到期状态，合同状态变更需在合同台账中单独操作。
- 工作台各项数据均为实时汇总，不落地存储，每次进入工作台时重新计算。

**边界与异常**

- **空数据**：关键指标、待办提醒、看板图表、最近动态任一区块无数据时，该区块显示"暂无数据"。
- **权限不足**：用户对某模块无查看权限时，该模块对应的关键指标、待办提醒、看板图表、快捷入口按权限展示，无权限部分不展示。

**（三）业务逻辑关联**

- 关键指标、看板图表数据来源于合同台账、履约节点、收付款计划的实时汇总，需相关模块已有数据方可展示统计结果。
- 待办提醒数据来源于站内提醒，站内提醒由系统扫描合同、履约节点、收付款计划的到期与逾期情况自动生成。
- 最近动态数据来源于合同台账的新增记录、履约节点的完成记录、收付款计划的完成记录。
- 快捷入口跳转至合同台账、履约提醒、收付款管理等对应功能模块，用户需具备目标模块的访问权限方可正常跳转。

### 3.5.2 合同管理

合同管理是系统的核心，涵盖合同的登记、归档、履约跟踪、收付款跟进、到期提醒和经营分析。合同台账负责合同的登记与查询，合同详情集中维护单份合同的附件、履约节点和收付款计划，履约提醒和收付款管理分别从履约与收付款维度集中跟进，经营分析提供整体经营视图。

#### 3.5.2.1 合同台账

**（一）功能需求概述**

合同台账是合同管理的主入口，用于集中查询、新增、编辑、查看和删除合同主记录，并支持按筛选结果导出。新增或编辑合同后，可在合同详情中继续维护附件、履约节点和收付款计划。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 合同名称 | 文本输入 | 支持模糊搜索 |
| 合同编号 | 文本输入 | 支持模糊搜索 |
| 合同类型 | 单选 | 枚举值：采购合同/销售合同/服务合同/租赁合同/框架协议/其他合同 |
| 合同状态 | 单选 | 枚举值：草稿/履行中/已到期/已终止/已归档 |
| 相对方 | 文本输入 | 支持模糊搜索 |
| 负责人 | 层级选择 | 来源于组织管理中已启用的用户 |
| 签订日期 | 日期范围 | 选择开始和结束日期 |
| 生效日期 | 日期范围 | 选择开始和结束日期 |
| 到期日期 | 日期范围 | 选择开始和结束日期 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 合同编号 | 文本 | - |
| 合同名称 | 文本 | - |
| 合同类型 | 文本 | - |
| 签约主体 | 文本 | - |
| 相对方 | 文本 | - |
| 合同金额 | 数字 | 保留2位小数 |
| 币种 | 文本 | - |
| 签订日期 | 日期 | - |
| 生效日期 | 日期 | - |
| 到期日期 | 日期 | - |
| 负责人 | 文本 | - |
| 合同状态 | 状态标签 | 草稿/履行中/已到期/已终止/已归档 |
| 操作 | 操作按钮 | 查看、编辑、删除（无权限时隐藏） |

分页默认每页10条，支持切换每页数量。列表为空时显示：暂无合同数据。

**新增功能**

点击"新增"，弹出编辑窗口，填写合同信息。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 合同编号 | 文本输入 | 2-50字 | 是 | - | 系统内全局唯一 |
| 合同名称 | 文本输入 | 2-100字 | 是 | - | - |
| 合同类型 | 单选 | - | 是 | - | 枚举值：采购合同/销售合同/服务合同/租赁合同/框架协议/其他合同 |
| 签约主体 | 文本输入 | 2-100字 | 是 | - | - |
| 相对方 | 文本输入 | 2-100字 | 是 | - | - |
| 合同金额 | 数字输入 | - | 是 | - | 非负数，保留2位小数 |
| 币种 | 单选 | - | 是 | 人民币 | 枚举值：人民币/美元/欧元/其他 |
| 签订日期 | 日期 | - | 是 | - | - |
| 生效日期 | 日期 | - | 是 | - | 不早于签订日期 |
| 到期日期 | 日期 | - | 是 | - | 不早于生效日期 |
| 负责人 | 层级选择 | - | 是 | - | 来源于组织管理中已启用的用户 |
| 合同状态 | 单选 | - | 是 | 草稿 | 枚举值：草稿/履行中/已到期/已终止/已归档 |
| 备注 | 多行文本输入 | 0-500字 | 否 | - | - |

提交成功：新增成功。提交失败：保留填写内容，显示具体错误原因。

**编辑功能**

点击"编辑"，自动填入当前记录数据，表单字段同新增。
提交成功：保存成功。提交失败：保留填写内容，显示具体错误原因。

**查看功能**

点击"查看"，进入合同详情，展示合同基础信息，并可在详情中继续维护该合同的附件、履约节点和收付款计划。

**删除功能**

点击"删除"，弹出确认"确认删除该合同及其关联数据吗？"。
删除成功：删除成功。
删除合同时，一并删除其关联的附件、履约节点、收付款计划和相关站内提醒，该合同的操作记录保留。

**导出功能**

点击"导出"，将当前筛选结果导出为文件。导出成功：导出成功。

**业务规则**

- 合同编号在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"合同编号【XXX】已存在，请更换"。
- 生效日期不早于签订日期，到期日期不早于生效日期。违反时阻止提交，提示"日期填写有误，请检查签订、生效、到期日期的先后顺序"。
- 合同到期后不自动变更为终止状态，系统仅提示到期情况，最终状态需由有权限人员确认维护。
- 合同金额仅用于记录与统计，不涉及核销、发票匹配或汇率换算。
- 合同数据对所有登录用户可见，新增、编辑、删除、导出为受控操作，无权限时对应操作入口隐藏。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无合同数据"。
- **权限不足**：新增、编辑、删除、导出按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **日期非法**：生效日期早于签订日期或到期日期早于生效日期时，阻止提交，提示"日期填写有误，请检查签订、生效、到期日期的先后顺序"。

**（三）业务逻辑关联**

- 负责人字段数据来源于组织管理中已启用的用户，需组织管理先完成人员维护。
- 合同主记录被附件、履约节点、收付款计划、站内提醒、操作记录引用，删除合同时一并级联删除上述关联数据（操作记录保留）。
- 合同主记录数据被工作台和经营分析引用，用于展示待办事项和统计分析。

#### 3.5.2.2 合同详情

点击"登记实际收付款"，弹出编辑窗口。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 实际金额 | 数字输入 | 保留2位小数 | 是 | - | ≥0 |
| 实际日期 | 日期 | - | 是 | - | - |

提交成功：保存成功，计划状态变为"已完成"。

**删除收付款计划**

点击"删除"，弹出确认"确认删除该收付款计划吗？"。
删除成功：删除成功。

**收付款计划状态流转规则**

- 新增计划默认状态为"待处理"。
- 计划日期已过且未登记实际收付款时，状态自动变为"逾期"。
- 登记实际收付款后，状态变为"已完成"。
- 部分收付款用实际金额与备注记录，不做拆分核销。

**操作记录**

分区展示当前合同的操作记录，只读，由系统自动生成，无新增、编辑、删除入口。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 操作类型 | 文本 | 枚举值：新增 / 编辑 / 删除 / 附件上传 / 节点完成 / 收付款完成 |
| 操作对象 | 文本 | - |
| 操作人 | 文本 | 系统自动记录 |
| 操作时间 | 日期时间 | 系统自动记录 |
| 操作说明 | 文本 | 无值显示"-" |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无操作记录"。

**业务规则**

- 计划日期临近的判定阈值默认取计划日期前30天，可由系统参数调整。
- 履约节点、收付款计划标记完成或登记实际收付款后，不再按计划日期判断逾期。
- 履约节点、收付款计划新增时默认状态均为"待处理"。
- 收付款计划的金额只做记录与统计，不做核销、发票匹配、汇率换算。
- 上传附件失败时不创建附件记录，提示"上传失败，请重试"。
- 删除附件仅删除附件记录，保留合同主记录及其他业务数据。
- 附件本期不做版本管理与在线预览，多版本文件通过文件名称、分类、备注加以区分。
- 履约节点、收付款计划、附件上传的关键操作完成后，系统自动写入操作记录。

**边界与异常**

- **空数据**：附件列表无数据时，显示"暂无附件数据"；履约节点列表无数据时，显示"暂无履约节点数据"；收付款计划列表无数据时，显示"暂无收付款计划数据"；操作记录无数据时，显示"暂无操作记录"。
- **权限不足**：上传、新增、编辑、标记完成、登记实际收付款、删除等操作按钮对无权限角色完全隐藏（不渲染，非禁用）。

**（三）业务逻辑关联**

- 履约节点标记完成、收付款计划登记实际收付款后，影响工作台的最近动态展示。
- 附件、履约节点、收付款计划的数据汇总影响工作台指标和经营分析的统计结果。
- 附件上传、履约节点标记完成、收付款计划登记实际收付款等关键操作自动写入操作记录。

#### 3.5.2.3 履约提醒

| 计划日期 | 日期 | 数据来源：合同到期日期或履约节点计划日期 |
| 逾期状态 | 状态标签 | 未逾期 / 已逾期，数据来源：系统根据计划日期与当前日期自动判断 |

- 本模块不新增独立业务实体，是合同到期与履约节点在提醒维度上的集中跟进视图

#### 3.5.2.5 经营分析

**（一）功能需求概述**

经营分析为管理者提供合同整体规模、金额分布、收付款进度和逾期风险的统计视图，帮助其快速掌握经营状况、发现风险点，为决策提供数据支撑。本功能为纯统计展示，数据实时汇总自合同台账、履约节点、收付款计划，不维护业务数据。

**（二）功能设计**

**条件筛选**

分析结果支持按以下条件筛选，条件之间为"与"关系，筛选变化后指标与图表同步刷新：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 日期范围 | 日期范围 | 按签订日期选择开始和结束日期 |
| 合同类型 | 下拉单选 | 枚举值：采购合同 / 销售合同 / 服务合同 / 租赁合同 / 框架协议 / 其他合同 |
| 合同状态 | 下拉单选 | 枚举值：草稿 / 履行中 / 已到期 / 已终止 / 已归档 |

**指标查看**

展示以下 7 项指标，均按当前筛选条件动态计算：

| 指标项 | 说明 |
|-------|------|
| 合同总数 | 筛选条件下的合同数量（份），来源于合同台账 |
| 合同总金额 | 筛选条件下合同金额之和（元），来源于合同台账 |
| 应收金额 | 筛选条件下合同关联收款计划的应收总额，来源于收付款计划中收付方向为"收款"的记录汇总 |
| 应付金额 | 筛选条件下合同关联付款计划的应付总额，来源于收付款计划中收付方向为"付款"的记录汇总 |
| 待收金额 | 应收金额中状态为"待处理"或"逾期"的收款计划金额汇总 |
| 待付金额 | 应付金额中状态为"待处理"或"逾期"的付款计划金额汇总 |
| 逾期金额 | 收付款计划中状态为"逾期"的收款和付款金额汇总 |

**分布分析**

- **按合同类型查看数量分布**：展示筛选条件下各合同类型（采购合同/销售合同/服务合同/租赁合同/框架协议/其他合同）的合同数量占比。
- **按合同状态查看数量分布**：展示筛选条件下各合同状态（草稿/履行中/已到期/已终止/已归档）的合同数量占比。

**趋势分析**

- **签约金额变化趋势**：按月份查看筛选条件下签约合同金额的变化趋势，每个月份对应该月签约合同的金额汇总。

**收付款进度**

- **收付款计划完成情况**：展示筛选条件下收付款计划按完成状态（已完成 / 待处理 / 逾期）划分的数量或金额占比。
- **收付情况对比**：对比以下四组数据，帮助管理者掌握整体收付款风险。

| 对比项 | 说明 |
|-------|------|
| 应收金额 / 应付金额 | 收付款计划中收款、付款方向的计划金额汇总 |
| 已收金额 / 已付金额 | 已登记实际收款、实际付款的金额汇总 |
| 待收金额 / 待付金额 | 状态为"待处理"或"逾期"的收款、付款计划金额汇总 |
| 逾期金额 | 状态为"逾期"的收付款计划金额汇总 |

**业务规则**

- 所有指标和分析结果均基于筛选条件动态计算，筛选条件变化后，指标查看与各分析结果同步刷新。
- 逾期判断依据计划日期与完成状态：计划日期已过且状态未变更为"已完成"的收付款计划或履约节点判定为逾期；已标记完成的不再判定逾期。
- 即将到期的合同数量、逾期履约节点数量纳入风险类指标，作为经营风险的参考依据。
- 各项统计数据均为实时汇总计算，不进行落地存储。
- 经营分析数据对所有登录用户可查看。

**边界与异常**

- **空数据**：筛选条件下无符合数据时，指标查看、分布分析、趋势分析、收付款进度各区块均显示"暂无数据"。
- **权限不足**：无经营分析模块访问权限的角色，对应菜单入口完全隐藏（不渲染，非禁用）。

**（三）业务逻辑关联**

- 指标查看、分布分析、趋势分析的数据来源于合同台账中的合同记录，按筛选条件动态汇总。
- 收付款进度数据来源于收付款计划的完成状态和金额，逾期判断同时参考履约节点的计划日期与完成状态。
- 经营分析与工作台看板共享同一数据来源，但提供更完整的多维统计和筛选能力，工作台看板仅展示概览性指标。

### 3.5.3 系统配置

系统配置复用企业已有管理平台的基础能力，为合同管理提供用户、角色、权限、组织和基础数据支撑，仅系统管理员可维护。本章节对各模块作功能概述，字段与交互沿用平台既有规范，不再展开完整字段设计。

#### 3.5.3.1 用户管理

**（一）功能需求概述**

用户管理用于维护系统登录账号及其所属部门、岗位和角色，是权限控制的基础。

**（二）功能设计**

- **列表与筛选**：分页查看用户，支持按用户名、姓名、手机号、状态筛选。列表为空时显示"暂无用户数据"。
- **新增/编辑**：维护用户的用户名、姓名、手机号、邮箱、所属部门、岗位和角色，用户名在系统内唯一。新增成功提示"新增成功"，编辑成功提示"保存成功"。
- **删除**：删除用户，弹出确认"确认删除该用户吗？"，删除成功提示"删除成功"。
- **重置密码**：将用户密码重置为默认密码，重置后原登录凭证失效，提示"密码已重置"。
- **启用/停用**：切换用户启用或停用状态，停用后该用户不能登录，切换成功提示"状态已更新"。

**业务规则**

- 用户名在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"用户名【XXX】已存在，请更换"。
- 用户停用后，该账号不能登录系统，已登录会话在凭证过期后失效。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无用户数据"。
- **权限不足**：新增、编辑、删除、重置密码、启用/停用操作仅系统管理员可见，其他角色不展示。

**（三）业务逻辑关联**

- 用户的部门来源于部门管理，岗位来源于岗位管理，角色来源于角色管理，需先完成相应基础数据维护。
- 用户被合同负责人、履约节点负责人等字段引用。

#### 3.5.3.2 角色管理

**（一）功能需求概述**

角色管理用于定义角色并为其分配菜单和操作权限，控制不同角色对合同各功能的访问范围。

**（二）功能设计**

- **列表**：分页查看角色。列表为空时显示"暂无角色数据"。
- **新增/编辑**：维护角色名称、角色标识和备注，角色标识在系统内唯一。新增成功提示"新增成功"，编辑成功提示"保存成功"。
- **删除**：删除角色，弹出确认"确认删除该角色吗？"，删除成功提示"删除成功"。
- **分配权限**：为角色勾选可访问的菜单和可执行的操作，保存后对该角色下所有用户生效，提示"保存成功"。

**业务规则**

- 角色标识在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"角色标识【XXX】已存在，请更换"。
- 删除角色时，若存在关联用户，阻止删除，提示"该角色下有关联用户，请先解除关联"。
- 角色权限变更后，对应角色的用户在下次进入系统时按新权限展示功能入口。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无角色数据"。
- **权限不足**：新增、编辑、删除、分配权限操作仅系统管理员可见。

**（三）业务逻辑关联**

- 角色被用户管理引用，用户通过角色获得功能权限。
- 角色权限决定 3.4 页面访问权限中各模块的受控操作可见性。

#### 3.5.3.3 部门管理

**（一）功能需求概述**

部门管理用于按层级维护企业组织结构，为用户归属和合同负责人选择提供组织数据。

**（二）功能设计**

- **列表**：按层级查看部门。列表为空时显示"暂无部门数据"。
- **新增/编辑**：维护部门名称、上级部门、负责人、联系电话和排序，支持启用或停用。新增成功提示"新增成功"，编辑成功提示"保存成功"。
- **删除**：删除部门，弹出确认"确认删除该部门吗？"，删除成功提示"删除成功"。

**业务规则**

- 删除部门时，若存在下级部门或关联用户，阻止删除，提示"该部门下有下级部门或关联用户，请先处理"。
- 部门停用后，不能被新增用户或合同负责人选择。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无部门数据"。
- **权限不足**：新增、编辑、删除操作仅系统管理员可见。

**（三）业务逻辑关联**

- 部门被用户管理引用，作为用户的组织归属。
- 合同负责人、履约节点负责人来源于组织管理中已启用的用户，用户按部门归类。

#### 3.5.3.4 岗位管理

**（一）功能需求概述**

岗位管理用于维护用户的职务归属维度，作为用户的岗位属性使用。

**（二）功能设计**

- **列表**：分页查看岗位。列表为空时显示"暂无岗位数据"。
- **新增/编辑**：维护岗位名称、岗位描述和排序，支持启用或停用，岗位名称在系统内唯一。新增成功提示"新增成功"，编辑成功提示"保存成功"。
- **删除**：删除岗位，弹出确认"确认删除该岗位吗？"，删除成功提示"删除成功"。

**业务规则**

- 岗位名称在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"岗位名称【XXX】已存在，请更换"。
- 岗位停用后，不能被新增用户选择。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无岗位数据"。
- **权限不足**：新增、编辑、删除操作仅系统管理员可见。

**（三）业务逻辑关联**

- 岗位被用户管理引用，作为用户的职务归属维度。

#### 3.5.3.5 字典管理

**（一）功能需求概述**

字典管理用于维护系统中可配置的枚举数据，如合同类型、附件分类等，便于统一管理业务基础数据。

**（二）功能设计**

- **字典类型**：查看、新增、编辑、删除字典类型，维护类型名称和类型键，类型键在系统内唯一。
- **字典项**：在字典类型下查看、新增、编辑、删除字典项，维护字典项名称、值和排序。
- 新增成功提示"新增成功"，编辑成功提示"保存成功"，删除弹出确认后提示"删除成功"。

**业务规则**

- 字典类型的类型键在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"类型键【XXX】已存在，请更换"。
- 删除字典类型时，一并删除其下所有字典项。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无字典数据"。
- **权限不足**：新增、编辑、删除操作仅系统管理员可见。

**（三）业务逻辑关联**

- 字典项为合同类型、合同状态、附件分类等枚举提供可配置数据来源。

#### 3.5.3.6 操作日志

**（一）功能需求概述**

操作日志记录系统关键操作，供管理员事后审计和问题追溯，只读展示。

**（二）功能设计**

- **列表**：分页查看操作日志，展示操作人、操作动作、请求 IP、归属地和操作时间。列表为空时显示"暂无操作日志数据"。
- **筛选**：支持按操作人、操作动作、操作时间筛选。
- 日志由系统自动记录，无新增、编辑、删除入口。

**业务规则**

- 操作日志由系统在关键操作发生时自动写入，用户不能手工修改或删除。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无操作日志数据"。
- **权限不足**：操作日志仅系统管理员可查看，其他角色不展示该菜单。

**（三）业务逻辑关联**

- 操作日志记录系统各模块的关键操作，与合同详情内的操作记录相互独立：操作日志面向全系统审计，合同操作记录聚焦单份合同。

## 四、技术方案

### 4.1 技术架构

系统采用前后端分离架构，在企业已有管理平台基础上扩展合同管理能力。

- **前端**：采用渐进式前端框架配合类型化脚本语言构建 PC 端管理后台，使用统一的企业级组件库保证界面一致性，使用图表库实现工作台看板与经营分析的可视化，使用文件导出能力支持合同台账和收付款计划的数据导出。
- **后端**：采用分层的服务端框架，通过数据访问层与关系型数据库交互，提供合同、附件、履约节点、收付款计划、提醒、操作记录等业务能力，并复用已有的用户、角色、部门、权限、日志等基础模块。
- **数据库**：采用关系型数据库存储合同主数据及其关联业务数据，合同附件文件存储于文件服务，数据库中仅保存附件的分类、名称、地址、大小等元数据。
- **定时任务**：通过服务端定时调度能力，周期性扫描合同到期日期、履约节点计划日期和收付款计划计划日期，生成或更新站内提醒。
- **部署**：前端与后端分别以容器方式部署，支持独立发布与横向扩展。

### 4.2 数据交互

- 前后端通过统一的数据接口交互，请求与响应采用标准的结构化数据格式。
- 列表查询统一支持分页、排序和多条件筛选，返回数据列表与分页信息。
- 关键操作（新增、编辑、删除、附件上传、标记完成、登记实际收付款）在处理成功后返回明确的结果状态和提示信息，失败时返回具体错误原因。
- 工作台指标、经营分析统计等汇总数据由服务端实时计算后返回，前端不重复维护业务数据。

### 4.3 安全方案

- **身份认证**：用户通过账号密码登录，登录后系统签发访问凭证，凭证过期或修改密码后失效。
- **权限控制**：基于角色的访问控制，合同数据对所有登录用户可见，新增、编辑、删除、导出、附件与计划维护等受控操作按角色权限校验，无权限时对应操作入口不展示。
- **数据传输**：数据传输使用加密通道，防止敏感信息在传输过程中泄露。
- **安全审计**：合同的新增、编辑、删除、附件上传、节点完成、收付款完成等关键操作记录操作人、操作时间和操作内容，支持事后追溯。

## 五、非功能需求

### 5.1 性能需求

- 页面加载时间小于 3 秒，常规接口响应时间小于 1 秒。
- 合同台账、收付款管理等列表在数据量增长时，分页查询响应保持稳定。
- 工作台与经营分析的汇总统计在合理数据规模下响应时间可接受。

### 5.2 安全需求

- 身份认证：账号密码登录，访问凭证过期或改密后失效。
- 数据安全：敏感数据加密存储，数据传输使用加密通道。
- 权限控制：基于角色的访问控制（RBAC），受控操作按角色权限校验。

### 5.3 可用性需求

- 系统可用性不低于 99.9%。
- 数据定期自动备份，支持故障后恢复。
- 关键操作提供明确的成功与失败反馈。

### 5.4 兼容性需求

- 浏览器：Chrome、Firefox、Safari、Edge 最新版本。
- 分辨率：适配 1920×1080、1366×768 等常见桌面分辨率。
