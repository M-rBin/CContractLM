# 需求背景与目标

## 需求背景

系统设置模块是企业内部合同管理系统的权限与基础数据底座，为合同管理提供账号、组织和权限支撑。当前存在以下痛点：

1. **账号与权限管理分散**：合同管理涉及业务负责人、财务、法务等多类角色，若账号和权限缺乏统一维护入口，容易出现权限过大或越权操作。
2. **组织与岗位数据缺失基准**：合同负责人、履约节点负责人均需从企业组织中选取，缺少统一的部门和岗位数据会导致人员归属混乱、选择困难。
3. **枚举数据硬编码难维护**：合同类型、附件分类等枚举若硬编码在系统中，业务调整时需改动代码，无法由管理员自助维护。
4. **关键操作缺乏审计留痕**：账号、权限、组织等敏感变更若无操作日志，事后难以追溯责任人和变更内容。

## 产品目标

1. **账号统一管理**：集中维护登录账号及其部门、岗位、角色归属，账号唯一性校验通过率 100%，杜绝重复账号。
2. **权限精确控制**：基于角色的访问控制（RBAC），角色权限变更后对该角色下全部用户下次进入系统时生效，权限配置覆盖全部功能模块。
3. **组织数据规范**：以层级结构维护部门，停用部门不可被新账号或合同负责人选择，组织数据可作为合同负责人选择的唯一来源。
4. **基础数据自助维护**：字典类型与字典项支持管理员自助增删改，业务枚举调整无需代码变更。
5. **操作全程可审计**：账号、权限、组织、字典等关键操作 100% 自动留痕，日志不可篡改，支持按操作人、动作、时间检索。

## 产品边界

| 纳入范围 | 排除范围（后续迭代规划）|
|---|---|
| 1. 用户管理（增删改查、重置密码、启停） | 1. 用户自助注册与找回密码 |
| 2. 角色管理与权限分配 | 2. 数据行级权限、字段级权限隔离 |
| 3. 部门层级管理 | 3. 组织架构导入导出、批量调岗 |
| 4. 岗位管理 | 4. 岗位与薪酬、绩效等 HR 能力联动 |
| 5. 字典类型与字典项管理 | 5. 多语言字典、字典版本管理 |
| 6. 操作日志查询 | 6. 日志告警、行为分析、外部审计对接 |

> **说明**：系统设置模块复用企业已有管理平台基础能力，字段与交互沿用平台既有规范。合同业务数据对所有登录用户可见，本模块权限控制的是各模块的受控操作（新增、编辑、删除等），不做合同数据的行级隔离。

### US-001：用户账号管理

**用户故事**
> 作为 **系统管理员**，
> 我想要 **新增、编辑、删除用户账号并为其分配部门、岗位和角色，同时可重置密码、启用或停用账号**，
> 以便 **企业员工能按各自角色登录并使用系统对应功能**。

**优先级**：P0（MVP）　　**估算**：3 人天　　**依赖**：US-002、US-003、US-004

**验收标准：**
- [ ] AC1：填写用户名、姓名、手机号、部门、岗位、角色并提交，新增成功后列表出现该用户，提示"新增成功"。
- [ ] AC2：用户名重复时阻止提交，提示"用户名【XXX】已存在，请更换"。
- [ ] AC3：对某用户执行重置密码，密码恢复为默认密码，原登录凭证失效，提示"密码已重置"。
- [ ] AC4：停用某用户后，该账号无法登录，已登录会话在凭证过期后失效；再次启用后可正常登录。
- [ ] AC5：删除用户前弹出确认"确认删除该用户吗？"，确认后删除成功，提示"删除成功"。

**关联功能需求**：见 3.1 用户管理　　**关联数据模型**：sys_user、sys_user_role

# 功能需求详述

> 系统设置模块划分为六个子模块：用户管理、角色管理、部门管理、岗位管理、字典管理、操作日志。仅系统管理员可维护，其他角色不展示对应菜单。以下接口路径为示意约定，字段与交互沿用平台既有规范。

## 用户管理

> **模块职责**：维护系统登录账号及其所属部门、岗位、角色，是权限控制的账号基础。

### 功能说明

- 列表查询：分页查看用户，支持按用户名、姓名、手机号、状态筛选，列表为空时显示"暂无用户数据"
- 新增用户：维护用户名、姓名、手机号、邮箱、所属部门、岗位、角色，用户名系统内唯一
  - 部门来源于部门管理已启用部门，岗位来源于岗位管理已启用岗位，角色来源于角色管理
- 编辑用户：修改用户信息，用户名唯一性校验规则同新增
- 删除用户：弹出确认后删除
- 重置密码：将密码重置为默认密码，重置后原登录凭证失效
- 启用/停用：切换启停状态，停用后不能登录

### 接口规格（API / 消息 / 事件）

#### 分页查询用户

**GET /system/user/page**

**Request:**
```json
{
  "username": "string，用户名，模糊匹配，可选",
  "nickName": "string，姓名，模糊匹配，可选",
  "phone": "string，手机号，可选",
  "status": "string，状态：0启用/1停用，可选",
  "pageNum": "int，页码，从1开始",
  "pageSize": "int，每页条数，默认10"
}
```

**Response (200):**
```json
{
  "code": 200,
  "total": "long，总条数",
  "rows": [
    {
      "userId": "long，用户ID",
      "username": "string，用户名",
      "nickName": "string，姓名",
      "phone": "string，手机号",
      "deptName": "string，所属部门名称",
      "postName": "string，岗位名称",
      "roleNames": "string，角色名称，多个逗号分隔",
      "status": "string，状态",
      "createTime": "string，创建时间"
    }
  ]
}
```

**错误码：**

| 错误码 | 含义 | 处理建议 |
|--------|------|---------|
| 401 | 未登录或凭证失效 | 跳转登录页 |
| 403 | 无用户管理权限 | 提示无权限并隐藏入口 |

**幂等性**：查询接口，天然幂等。

#### 新增用户

**POST /system/user**

**Request:**
```json
{
  "username": "string，用户名，必填，唯一",
  "nickName": "string，姓名，必填",
  "phone": "string，手机号，可选",
  "email": "string，邮箱，可选",
  "deptId": "long，所属部门ID，必填",
  "postIds": "long[]，岗位ID列表，可选",
  "roleIds": "long[]，角色ID列表，必填"
}
```

**Response (200):**
```json
{ "code": 200, "msg": "新增成功", "data": { "userId": "long，新用户ID" } }
```

**错误码：**

| 错误码 | 含义 | 处理建议 |
|--------|------|---------|
| 40001 | 用户名已存在 | 提示"用户名【XXX】已存在，请更换" |
| 40002 | 部门/岗位/角色不存在或已停用 | 提示重新选择 |

**幂等性**：非幂等，前端提交后置灰按钮防重复提交。

#### 编辑用户

**PUT /system/user**：请求体在新增基础上增加 `userId`，用户名唯一性校验排除自身。返回 `{"code":200,"msg":"保存成功"}`。

#### 删除用户

**DELETE /system/user/{userId}**：返回 `{"code":200,"msg":"删除成功"}`。

#### 重置密码

**PUT /system/user/resetPwd**

**Request:** `{ "userId": "long，必填" }`
**Response:** `{ "code": 200, "msg": "密码已重置" }`

**幂等性**：重复重置结果一致，幂等。

#### 启用/停用

**PUT /system/user/changeStatus**

**Request:** `{ "userId": "long", "status": "string，0启用/1停用" }`
**Response:** `{ "code": 200, "msg": "状态已更新" }`

**幂等性**：以目标状态为准，幂等。

### 业务规则

| 编号 | 规则 | 触发条件 | 处理方式 |
|------|------|---------|---------|
| BR-U01 | 用户名全局唯一 | 新增/编辑提交时 | 重复则阻止提交，提示"用户名【XXX】已存在，请更换" |
| BR-U02 | 停用禁止登录 | 用户状态为停用 | 该账号登录被拒，已登录会话凭证过期后失效 |
| BR-U03 | 归属数据有效性 | 新增/编辑选择部门、岗位、角色 | 所选部门/岗位须为启用状态，角色须存在，否则阻止提交 |
| BR-U04 | 密码重置失效旧凭证 | 执行重置密码 | 重置为默认密码，原访问凭证立即失效 |

### 数据模型

**sys_user（用户表）**

| 字段名称 | 数据类型 | 字段长度 | 是否必填 | 备注（业务说明）|
|---|---|---|---|---|
| user_id | BIGINT | — | 是 | 主键，用户ID |
| username | VARCHAR | 30 | 是 | 用户名，全局唯一 |
| nick_name | VARCHAR | 30 | 是 | 姓名 |
| password | VARCHAR | 100 | 是 | 密码，加密存储 |
| phone | VARCHAR | 20 | 否 | 手机号 |
| email | VARCHAR | 50 | 否 | 邮箱 |
| dept_id | BIGINT | — | 是 | 所属部门ID，关联 sys_dept |
| post_id | BIGINT | — | 否 | 岗位ID，关联 sys_post |
| status | CHAR | 1 | 是 | 状态：0启用/1停用，默认0 |
| create_time | DATETIME | — | 是 | 创建时间 |
| update_time | DATETIME | — | 否 | 更新时间 |

**sys_user_role（用户角色关联表）**

| 字段名称 | 数据类型 | 字段长度 | 是否必填 | 备注（业务说明）|
|---|---|---|---|---|
| user_id | BIGINT | — | 是 | 用户ID，联合主键 |
| role_id | BIGINT | — | 是 | 角色ID，联合主键，关联 sys_role |

## 角色管理

> **模块职责**：定义角色并为其分配菜单和操作权限，控制不同角色对系统各功能的访问范围。

### 功能说明

- 列表查询：分页查看角色，列表为空时显示"暂无角色数据"
- 新增角色：维护角色名称、角色标识、备注，角色标识系统内唯一
- 编辑角色：修改角色信息，角色标识唯一性校验排除自身
- 删除角色：弹出确认后删除，存在关联用户时阻止删除
- 分配权限：为角色勾选可访问菜单和可执行操作，保存后对该角色下所有用户生效

### 接口规格（API / 消息 / 事件）

#### 分页查询角色

**GET /system/role/page**

**Request:**
```json
{
  "roleName": "string，角色名称，模糊匹配，可选",
  "roleKey": "string，角色标识，可选",
  "pageNum": "int，页码",
  "pageSize": "int，每页条数"
}
```

**Response (200):**
```json
{
  "code": 200,
  "total": "long",
  "rows": [
    {
      "roleId": "long，角色ID",
      "roleName": "string，角色名称",
      "roleKey": "string，角色标识",
      "remark": "string，备注",
      "createTime": "string，创建时间"
    }
  ]
}
```

# 状态机

> 系统设置模块的核心状态对象为"用户账号"，其余对象（部门、岗位）仅有启用/停用两态，一并列出。

## 用户账号状态机

| 状态 | 触发动作 | 触发方 | 下一状态 | 后续操作/副作用 |
|---|---|---|---|---|
| （无） | 新增用户 | 系统管理员 | 启用 | 生成账号，默认启用，可登录 |
| 启用 | 停用 | 系统管理员 | 停用 | 账号不能登录，已登录会话凭证过期后失效 |
| 停用 | 启用 | 系统管理员 | 启用 | 账号恢复登录能力 |
| 启用/停用 | 重置密码 | 系统管理员 | 状态不变 | 密码恢复默认，原访问凭证立即失效 |
| 启用/停用 | 删除 | 系统管理员 | （已删除） | 账号删除，其角色关联一并解除 |

## 部门 / 岗位启停状态机

| 状态 | 触发动作 | 触发方 | 下一状态 | 后续操作/副作用 |
|---|---|---|---|---|
| （无） | 新增 | 系统管理员 | 启用 | 默认启用，可被账号/合同负责人选择 |
| 启用 | 停用 | 系统管理员 | 停用 | 不再被新增账号或合同负责人选择，已引用数据不受影响 |
| 停用 | 启用 | 系统管理员 | 启用 | 恢复可选 |
| 启用/停用 | 删除 | 系统管理员 | （已删除） | 部门存在下级或关联用户时阻止删除；岗位可直接删除 |

# 业务主流程

系统管理员搭建权限基础数据的端到端主流程：

```
新增部门（管理员）--> 新增岗位（管理员）--> 新增角色（管理员）
  --> 为角色分配菜单和操作权限（管理员）
  --> 新增用户并绑定部门/岗位/角色（管理员）
  --> 用户使用账号密码登录（用户，系统校验状态与凭证）
  --> 系统按用户所属角色权限展示功能入口（系统自动）
  --> 各模块关键操作自动写入操作日志（系统自动）
  --> 管理员按需查询操作日志进行审计（管理员）
```

字典维护为并行独立流程：

```
新增字典类型（管理员）--> 在类型下新增字典项（管理员）
  --> 字典项为合同类型、附件分类等枚举提供可配置数据来源（系统自动引用）
```

# 核心业务规则

> 以下为跨模块的全局规则，各模块内的规则见对应模块的业务规则表，不在此重复。

| 编号 | 规则 | 触发条件 | 处理方式 |
|------|------|---------|---------|
| BR-G01 | 系统设置仅管理员可维护 | 访问系统设置各模块 | 仅系统管理员展示菜单与受控操作，其他角色菜单入口完全隐藏（不渲染，非禁用） |
| BR-G02 | 唯一性校验统一话术 | 用户名/角色标识/岗位名称/类型键重复 | 阻止提交并提示"【字段值】已存在，请更换" |
| BR-G03 | 停用即不可选 | 部门/岗位停用后 | 停用对象不再出现在新增账号或合同负责人的选择范围，已引用的历史数据不受影响 |
| BR-G04 | 删除关联保护 | 删除角色/部门时 | 存在关联用户或下级时阻止删除，先解除关联再删除 |
| BR-G05 | 权限变更下次登录生效 | 角色权限保存后 | 该角色下用户下次进入系统按新权限展示，不强制中断在线会话 |
| BR-G06 | 关键操作强制留痕 | 增删改、权限分配、重置密码、启停成功后 | 系统自动写入操作日志，日志不可篡改 |

## 权限判定规则

> 判定逻辑：用户 → 角色（sys_user_role）→ 菜单/操作权限（sys_role_menu）。
>
> - 用户拥有的权限 = 其绑定的所有角色权限的并集。
> - 某功能入口是否展示：`用户权限集合 ∩ 该功能所需权限 ≠ 空` 则展示，否则隐藏。
> - 边界情况：用户未绑定任何角色时，权限集合为空，仅可见默认公共页面；停用用户不进入权限判定，直接拒绝登录。

## 唯一性校验规则

> 校验范围：用户名（sys_user.username）、角色标识（sys_role.role_key）、岗位名称（sys_post.post_name）、字典类型键（sys_dict_type.dict_type）均为全局唯一。
>
> - 新增时：全表校验重复。
> - 编辑时：校验重复但排除记录自身。
> - 边界情况：空值不参与唯一性判断，由必填校验拦截。

# 非功能需求

1. **性能需求**：列表分页查询 P99 响应时间 < 1 秒；部门树在千级节点内加载 < 2 秒；权限校验在登录及页面进入时完成，单次判定 < 100ms。
2. **兼容性需求**：支持 Chrome、Firefox、Safari、Edge 最新版本；适配 1920×1080、1366×768 常见桌面分辨率。
3. **安全性需求**：密码加密存储，不可逆；操作日志不可篡改、不可删除；数据传输使用加密通道；停用与删除账号即时阻断登录能力；请求参数入库前对密码等敏感字段脱敏。
4. **易用性需求**：所有增删改操作步骤 ≤ 3 步；删除操作统一二次确认；表单校验失败保留已填内容并提示具体原因；成功/失败均有明确反馈文案。

# 验收标准

- [ ] **功能完整性**：用户、角色、部门、岗位、字典、操作日志六个子模块的增删改查及受控操作（重置密码、启停、分配权限）均可正常使用，无功能缺失。
- [ ] **唯一性校验准确**：用户名、角色标识、岗位名称、类型键重复时 100% 拦截，提示话术符合规范。
- [ ] **权限控制正确**：非管理员角色不展示系统设置菜单；角色权限变更后该角色用户下次进入按新权限展示功能入口。
- [ ] **关联保护有效**：删除存在关联用户的角色、存在下级或关联用户的部门时被正确阻止并提示。
- [ ] **停用生效**：停用用户不能登录；停用部门、岗位不出现在新增账号选择范围。
- [ ] **日志不可篡改**：操作日志仅可查询，无任何修改、删除入口，关键操作 100% 留痕。
- [ ] **性能达标**：满足第 8 章非功能需求中规定的各项指标。
