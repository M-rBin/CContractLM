# 需求背景与目标

## 需求背景

企业日常合同长期散落在个人电脑、聊天记录和线下表格中，签署后的执行跟踪、到期提醒和收付款跟进主要依赖人工，存在以下痛点：

1. **合同信息分散无台账**：合同正文、金额、期限、负责人等关键信息缺乏统一登记入口，查询和统计困难。
2. **履约节点易遗漏**：交付、验收、续签、终止等关键事项缺少跟踪机制，临期和逾期无人提醒。
3. **收付款进度不透明**：应收、应付、待收、待付、逾期金额无法集中掌握，资金跟进滞后。
4. **附件资料难归档**：合同正文、补充协议、验收材料、发票、付款凭证等材料缺乏分类归档位置。
5. **经营状况缺乏概览**：管理者无法快速掌握合同规模、金额分布、收付款进度和逾期风险。

## 产品目标

1. **统一合同台账**：集中登记合同基础信息，支持多维度查询、筛选与导出，减少信息散落。
2. **附件分类归档**：按业务材料类型对合同附件分类上传、查看、下载与删除。
3. **履约节点跟踪**：维护合同的交付、验收、续签、终止等履约节点，临期与逾期自动提醒。
4. **收付款计划管理**：按收/付方向维护收付款计划，形成待处理、已完成、逾期的状态跟踪。
5. **站内提醒驱动**：系统扫描合同到期、履约节点计划日期、收付款计划日期，自动生成提醒并支持跳转处理。
6. **工作台总览**：登录后默认进入合同工作台，集中展示关键指标、待办提醒、看板图表与快捷入口。
7. **经营分析支撑**：为管理者提供合同规模、金额分布、收付款进度和逾期风险的多维统计视图。

## 产品边界

| 纳入范围 | 排除范围（后续迭代规划）|
|---|---|
| 1. 合同工作台（关键指标、待办提醒、看板图表、快捷入口、最近动态） | 1. 线上审批流、合同在线编辑、电子签章 |
| 2. 合同台账（增删改查、筛选、导出） | 2. 合同数据行级权限隔离（第一版对所有登录用户可见） |
| 3. 合同详情（基础信息、附件、履约节点、收付款计划、操作记录） | 3. 附件版本管理、在线预览强依赖 |
| 4. 履约提醒（合同到期与履约节点提醒的集中跟进） | 4. 收付款核销、发票匹配、汇率换算 |
| 5. 收付款管理（跨合同收付款计划集中查询与登记） | 5. 财务系统对接、银行流水导入 |
| 6. 经营分析（合同规模、金额分布、收付款进度、逾期风险统计） | 6. 自定义报表、数据导出模板配置 |

> **说明**：合同业务复用系统设置模块提供的用户、角色、部门、岗位、权限、日志等基础能力（详见系统设置模块 PRD）。合同数据第一版对所有登录用户可见，本模块权限控制的是各功能的受控操作（新增、编辑、删除、导出、上传、登记等）。

### US-001：合同工作台总览

**用户故事**
> 作为 **系统使用者**，
> 我想要 **登录后在工作台集中查看合同关键指标、待办提醒、看板图表、快捷入口和最近动态**，

**优先级**：P0（MVP）　　**估算**：3 人天　　**依赖**：US-002~007

**验收标准：**
- [ ] AC1：登录后默认进入合同工作台，展示合同总数、履行中、即将到期、逾期、待收金额、待付金额、逾期金额 7 项关键指标。
- [ ] AC2：展示合同到期、履约节点、收款计划、付款计划 4 类待办提醒，点击可跳转至对应处理位置。
- [ ] AC3：展示合同状态分布、合同类型分布、月度签约金额、收付款完成情况 4 项看板图表。
- [ ] AC4：任一区块无数据时显示"暂无数据"，无权限模块对应内容不展示。

**关联功能需求**：见 3.1 合同工作台

### US-002：合同台账管理

**用户故事**
> 作为 **业务负责人**，
> 我想要 **登记、编辑、查询、删除并按筛选结果导出合同**，
> 以便 **集中管理所有合同的基础信息，减少信息散落**。

**优先级**：P0（MVP）　　**估算**：4 人天　　**依赖**：-

**验收标准：**
- [ ] AC1：填写合同编号、名称、类型、签约主体、相对方、金额、币种、签订/生效/到期日期、负责人、状态并提交，新增成功后列表出现该合同，提示"新增成功"。

- [ ] AC4：支持按名称、编号、类型、状态、相对方、负责人、签订/生效/到期日期范围筛选。
- [ ] AC5：删除合同二次确认后，一并删除其附件、履约节点、收付款计划和相关提醒，操作记录保留。

# 功能需求详述

> 合同管理模块包含合同工作台、合同台账、合同详情、履约提醒、收付款管理、经营分析六个功能。接口路径、字段名为前瞻设计，接口前缀 `admin/contract/`，后续以详细设计与实际实现为准。合同数据对所有登录用户可见，各功能的受控操作通过角色权限（权限标识 `contract:*`）控制。

## 合同工作台

> **模块职责**：登录后的默认首页，集中展示合同履行、收付款相关的关键数据和待办事项，帮助用户快速掌握业务状况并直达高频操作入口。工作台不维护业务数据，所有内容实时汇总自合同台账、履约节点、收付款计划、站内提醒。

### 功能说明

- 关键指标：展示合同总数、履行中合同、即将到期合同、逾期合同、待收金额、待付金额、逾期金额 7 项
- 待办提醒：展示合同到期、履约节点、收款计划、付款计划 4 类待办，点击跳转至对应处理位置
- 看板图表：展示合同状态分布、合同类型分布、月度签约金额、收付款完成情况 4 项
- 快捷入口：新增合同、合同台账、履约提醒、收付款管理 4 个入口
- 最近动态：展示最近新增合同、最近完成履约节点、最近完成收付款记录，点击跳转至对应合同详情

### 接口规格（API / 消息 / 事件）

#### 获取工作台汇总数据

**GET /admin/contract/workbench/overview**

**Response (200):**
```json
{
  "code": 200,
  "data": {
    "metrics": {
      "totalCount": "int，合同总数",
      "performingCount": "int，履行中合同数",
      "expiringCount": "int，即将到期合同数",
      "overdueCount": "int，逾期合同数",
      "receivableAmount": "number，待收金额",
      "payableAmount": "number，待付金额",
      "overdueAmount": "number，逾期金额"
    },
    "charts": {
      "statusDist": "[{ status, count }]，合同状态分布",
      "typeDist": "[{ type, count }]，合同类型分布",
      "monthlyAmount": "[{ month, amount }]，月度签约金额",
      "paymentProgress": "[{ status, count, amount }]，收付款完成情况"
    }
  }
}
```

**幂等性**：查询接口，天然幂等。

## 合同详情

> **模块职责**：从合同台账进入，集中展示与维护单个合同的基础信息、附件、履约节点、收付款计划和操作记录，为合同全生命周期的履约跟踪、资料留存和资金收付管理提供统一入口。

### 功能说明

- 基础信息：分区展示合同 13 项基础字段，支持编辑（字段同新增合同）
- 合同附件：按分类上传、查看、下载、删除附件
- 履约节点：新增、编辑、标记完成、删除履约节点
- 收付款计划：新增、编辑、登记实际收付款、删除收付款计划
- 操作记录：只读展示合同关键操作，由系统自动生成

### 接口规格（API / 消息 / 事件）

#### 合同附件

- **GET /admin/contract/attachment/list/{contractId}**：查询合同附件列表，返回分类、文件名称、文件大小、上传人、上传时间

## 履约提醒

> **模块职责**：集中展示即将到期、已逾期、待完成的合同到期与履约节点相关站内提醒，帮助用户及时跟进履约事项。本页聚焦合同到期和履约节点提醒（收付款提醒在收付款管理中处理），提醒数据由系统定期扫描自动生成，用户可从提醒直接跳转来源并完成处理。

### 功能说明

- 列表查询：分页查看提醒，支持按合同名称、提醒类型、逾期状态、处理状态、计划日期范围筛选
- 查看来源：跳转至提醒对应的来源合同详情或履约节点
- 完成处理：对来源履约节点标记完成，处理后提醒处理状态自动更新为"已处理"

**PUT /admin/contract/reminder/handle**

**Request:** `{ "id": "int，提醒ID", "actualDate": "string，实际完成日期，必填" }`

**幂等性**：以目标状态为准，重复处理返回已处理，幂等。

### 业务规则

| 编号 | 规则 | 触发条件 | 处理方式 |
|------|------|---------|---------|
| BR-R01 | 临期生成提醒 | 履约节点计划日期临近（默认前30天，可配） | 系统生成或更新对应履约节点提醒，处理状态"待处理" |
| BR-R02 | 逾期标记 | 履约节点超计划日期未完成 | 对应提醒逾期状态标记"已逾期" |
| BR-R03 | 完成自动同步 | 来源履约节点标记完成 | 提醒处理状态自动更新"已处理"，已处理不再参与逾期判断 |
| BR-R04 | 生成失败不阻断 | 提醒生成失败 | 不影响合同主记录和履约节点的正常维护 |
| BR-R05 | 完成处理权限 | 点击完成处理 | 受控操作，无权限时入口隐藏 |

### 边界与异常

- **空数据**：列表无数据时，显示"暂无履约提醒数据"。
- **权限不足**："完成处理"按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **来源数据缺失**：来源合同或履约节点已删除时，该提醒不再展示。

### 业务逻辑关联

- 提醒数据来源于合同主记录的到期日期和履约节点的计划日期、状态，需合同主记录和履约节点先完成维护。
- 完成处理会更新履约节点完成状态，同步影响工作台待办事项和最近动态。
- 本模块不新增独立业务实体（提醒记录为系统扫描生成的派生数据），是合同到期与履约节点在提醒维度上的集中跟进视图。

## 收付款管理

> **模块职责**：从收款计划和付款计划维度集中查询、跟进所有合同的收付款计划，为财务和业务人员提供跨合同的收付款集中处理视图。本模块不重复维护收付款计划业务数据，计划的新增和编辑在合同详情内完成。

### 功能说明

- 列表查询：按收款计划、付款计划两个维度分别分页展示，支持按合同名称、收付方向、状态、计划日期范围、相对方筛选
- 登记实际收付款：填写实际金额和实际日期，登记后状态变"已完成"
- 统计：展示应收、应付、待收、待付、逾期 5 项汇总金额
- 导出：将当前筛选结果导出为文件

### 业务逻辑关联

- 收付款计划数据来源于各合同详情内维护的收款计划和付款计划，本模块不新增业务实体，仅提供集中查询和登记入口。
- 登记实际收付款后，状态变更同步影响工作台待收/待付/逾期金额指标、最近动态以及经营分析的收付款统计。
- 查看来源合同跳转至对应合同详情，依赖合同主记录仍存在。

### 数据模型

复用合同详情的 **contract_payment_plan（收付款计划表）**，本模块不新增数据表。

## 经营分析

> **模块职责**：为管理者提供合同整体规模、金额分布、收付款进度和逾期风险的统计视图，帮助其快速掌握经营状况、发现风险点。本功能为纯统计展示，数据实时汇总自合同台账、履约节点、收付款计划，不维护业务数据。

### 功能说明

- 条件筛选：支持按日期范围（签订日期）、合同类型、合同状态筛选，筛选变化后指标与图表同步刷新
- 指标查看：展示合同总数、合同总金额、应收、应付、待收、待付、逾期 7 项指标
- 分布分析：按合同类型、合同状态查看数量分布
- 趋势分析：按月份查看签约金额变化趋势
- 收付款进度：查看收付款计划完成情况与收付情况对比

# 状态机

> 合同管理模块具备状态流转的对象为「合同」「履约节点」「收付款计划」「履约提醒」。合同状态由有权限人员手动维护；履约节点、收付款计划的逾期为系统自动判定、完成为手动操作；提醒处理状态随来源节点完成自动同步。

## 合同状态机

| 状态 | 触发动作 | 触发方 | 下一状态 | 后续操作/副作用 |
|---|---|---|---|---|
| （无） | 新增合同 | 有权限人员 | 草稿 | 生成合同主记录，默认草稿 |
| 草稿 | 编辑状态为履行中 | 有权限人员 | 履行中 | 合同进入履行跟踪 |
| 履行中 | 到达到期日期 | 系统 | 已到期（提示） | 工作台/提醒展示到期，状态需人工确认变更 |
| 履行中/已到期 | 编辑状态为已终止 | 有权限人员 | 已终止 | 合同终止，停止履约跟进 |
| 履行中/已到期/已终止 | 编辑状态为已归档 | 有权限人员 | 已归档 | 合同归档留存 |
| 任意状态 | 删除合同 | 有权限人员 | （已删除） | 级联删除附件、节点、计划、提醒，操作记录保留 |

## 履约节点状态机

| 状态 | 触发动作 | 触发方 | 下一状态 | 后续操作/副作用 |
|---|---|---|---|---|
| （无） | 新增节点 | 有权限人员 | 待处理 | 默认待处理 |
| 待处理 | 计划日期已过未完成 | 系统 | 逾期 | 生成/更新逾期提醒 |
| 待处理/逾期 | 标记完成 | 有权限人员 | 已完成 | 记录实际完成日期，同步提醒为已处理，不再判逾期 |
| 任意状态 | 删除节点 | 有权限人员 | （已删除） | 对应提醒不再展示 |

## 收付款计划状态机

| 状态 | 触发动作 | 触发方 | 下一状态 | 后续操作/副作用 |
|---|---|---|---|---|
| （无） | 新增计划 | 有权限人员 | 待处理 | 默认待处理 |
| 待处理 | 计划日期已过未登记 | 系统 | 逾期 | 纳入逾期金额统计 |
| 待处理/逾期 | 登记实际收付款 | 有权限人员 | 已完成 | 记录实际金额与日期，不再判逾期 |
| 任意状态 | 删除计划 | 有权限人员 | （已删除） | 从收付款管理与统计中移除 |

## 履约提醒处理状态机

| 状态 | 触发动作 | 触发方 | 下一状态 | 后续操作/副作用 |
|---|---|---|---|---|
| （无） | 计划日期临近 | 系统 | 待处理 | 生成或更新提醒 |
| 待处理 | 来源节点标记完成 / 完成处理 | 系统 / 有权限人员 | 已处理 | 不再参与逾期判断 |

# 业务主流程

合同从登记到履约跟进的端到端主流程：

```
登记合同（有权限人员，填写基础信息）--> 归档附件（分类上传合同正文/发票等）
  --> 维护履约节点（交付/验收/续签等，设定计划日期）
  --> 维护收付款计划（收款/付款方向，设定计划金额与日期）
  --> 系统扫描到期与临期，生成站内提醒（系统自动）
  --> 履约提醒集中跟进：查看来源、完成处理履约节点（有权限人员）
  --> 收付款管理集中跟进：登记实际收付款（财务人员）
  --> 工作台总览关键指标与待办、经营分析多维统计（管理者）
```

# 核心业务规则

> 以下为跨模块的全局规则，各模块内的规则见对应模块的业务规则表，不在此重复。

| 编号 | 规则 | 触发条件 | 处理方式 |
|------|------|---------|---------|
| BR-G01 | 合同数据全员可见 | 访问合同各模块 | 合同数据对所有登录用户可见，新增/编辑/删除/导出/上传/登记为受控操作 |
| BR-G02 | 唯一性交后端校验 | 合同编号等唯一字段重复 | 由后端返回错误信息，前端拦截器统一提示 |
| BR-G03 | 删除统一二次确认 | 任意删除操作 | 弹出确认框，确认后执行，成功提示"删除成功" |

| BR-G06 | 逾期综合判定 | 计算逾期状态 | 基于计划日期与完成状态判定，已完成不再判逾期 |
| BR-G07 | 金额不做核销 | 记录合同与收付款金额 | 金额仅用于记录与统计，不涉及核销、发票匹配、汇率换算 |
| BR-G08 | 负责人来源组织 | 选择合同/节点负责人 | 来源于系统设置模块中已启用的用户 |

# 非功能需求

1. **性能需求**：列表分页查询 P99 响应时间 < 1 秒；工作台与经营分析的汇总统计在万级合同数据内加载 < 2 秒。
2. **兼容性需求**：支持 Chrome、Firefox、Safari、Edge 最新版本；适配 1920×1080、1366×768 常见桌面分辨率。
3. **安全性需求**：合同数据传输使用加密通道；受控操作按角色权限控制，无权限入口隐藏；附件上传校验文件大小（单文件最大 50MB）。
4. **可靠性需求**：提醒生成失败不影响合同主记录和履约节点的正常维护；删除合同的级联删除在同一事务内完成，保证数据一致性。
5. **易用性需求**：删除操作统一二次确认；表单校验失败保留已填内容并内联提示；成功/失败均有明确反馈文案；请求错误由拦截器统一提示。

# 验收标准

- [ ] **功能完整性**：合同工作台、合同台账、合同详情、履约提醒、收付款管理、经营分析六个功能的查询、维护及受控操作均可正常使用。
- [ ] **合同台账**：支持多条件筛选；合同编号唯一校验生效；日期先后校验生效；删除级联清理关联数据、保留操作记录；筛选结果可导出。
- [ ] **合同详情**：基础信息可编辑；附件按分类上传/查看/下载/删除；履约节点新增/编辑/标记完成/删除，逾期自动判定；收付款计划新增/编辑/登记/删除，逾期自动判定；操作记录自动生成。
- [ ] **履约提醒**：临期与逾期提醒自动生成；查看来源可跳转；完成处理后来源节点标记完成、提醒状态同步为已处理。
- [ ] **收付款管理**：按收款/付款维度分列展示；登记实际收付款生效；应收/应付/待收/待付/逾期统计准确；筛选结果可导出。
- [ ] **合同工作台**：7 项关键指标、4 类待办、4 项看板、快捷入口、最近动态展示正确，待办可跳转。
- [ ] **经营分析**：筛选联动刷新；7 项指标、类型/状态分布、月度趋势、收付款进度与对比展示正确。
- [ ] **性能达标**：满足第 8 章非功能需求中规定的各项指标。
