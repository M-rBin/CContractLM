### 执行中发现并修复的问题

1. **附件下载 500 错误**：`ContractAttachmentController.download` 用裸 `@Res()` 但仍返回 `pipe(res)` 结果，触发全局拦截器对 Response 对象的循环引用递归，导致 500。已修复为不返回值 + 补充流错误处理，经 code-reviewer 两轮审查通过。
2. **CSV 导出内容不可用**：全局 `TransformInterceptor` 统一包装 JSON 导致 CSV 导出接口（合同台账导出、收付款管理导出，以及既有的用户导出）返回的文件内容是 JSON 字符串而非表格数据。已在拦截器层面修复（识别 `@Header` 显式声明的非 JSON Content-Type 并跳过包装），经 code-reviewer 审查通过（`DONE_WITH_CONCERNS`，采纳其 1 项 MEDIUM 建议做了加固）。

详见 `docs/contract-management-delivery-plan.md` 第四节「测试执行发现」。
