## 三、引言

### 3.1 编写目的

本文档用于指导企业内部合同管理系统系统设置模块的功能测试、联调验收和上线前回归测试，确保用户、部门、岗位、角色、菜单、字典、操作日志以及用户导入导出和批量设置能力符合需求说明书与交付计划要求。

### 3.2 测试范围

**Web 管理后台：**

- 用户管理：用户列表、新增、编辑、删除、批量删除、启用禁用、部门筛选、角色与岗位维护。
- 部门管理：部门树展示、新增下级、编辑、删除、部门类型与排序维护。
- 岗位管理：岗位列表、新增、编辑、删除、排序维护。
- 角色管理：角色列表、新增、编辑、删除、状态筛选、菜单权限分配。
- 菜单管理：菜单树展示、目录/菜单/按钮维护、显示隐藏。
- 字典管理：字典类型与字典项的查询、新增、编辑、删除。
- 操作日志：日志只读查询、筛选、详情查看。
- 用户导入导出：Excel 导入、CSV 导出。
- 用户批量设置：批量设置岗位、批量设置角色。

### 7.2 覆盖率说明

- **业务功能测试**：覆盖系统设置模块的列表查询、新增、编辑、删除、启用禁用、显示隐藏、权限分配、导入导出和批量设置。
- **异常测试**：覆盖必填字段为空、格式非法、唯一性冲突、删除已关联数据、接口异常和只读约束。
- **性能测试**：覆盖用户、部门、岗位、角色、菜单、字典、日志列表以及导入导出、批量设置等关键操作。
- **权限测试**：覆盖无权限菜单入口、按钮可见性和直接调用接口的权限拦截。

### 7.3 测试执行建议

1. **优先级 P0**：用户管理、角色管理、菜单管理、部门管理、岗位管理、字典管理。
2. **优先级 P1**：操作日志、用户导入导出、用户批量设置岗位/角色。
3. **回归范围**：每次修改权限、菜单或用户角色逻辑后，必须回归用户登录、菜单可见性、按钮权限和角色权限分配。
4. **缺陷记录**：缺陷需记录复现步骤、测试账号、请求接口、期望结果、实际结果和截图。
5. **上线门禁**：`admin npm run build` 与 `server npm run build` 必须通过；P0 缺陷必须全部关闭。
